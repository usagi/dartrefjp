Tutorial: Dart Editor (Windows)

Follow this tutorial to install and use Dart Editor on Windows.

Dart Editor is a lightweight, open-source editor based on Eclipse components. In addition to editing Dart programs, you can use Dart Editor to invoke the Dart-to-JavaScript compiler and launch Dart-based web apps.

Note: These instructions are also available for:
  • Mac
  • Linux

Step 1: Download and install the software

In this step, you'll install Dart Editor and, if necessary, a Java runtime environment.

  1. Download the Dart Editor ZIP file that matches your system architecture:
    • 32-bit Dart Editor
    • 64-bit Dart Editor

    If you don't know what your system architecture is, see Microsoft's support site.

    Version note: For information about this Dart Editor version, see the change log. If you want to try out the most recent, untested version of Dart Editor, use one of these ZIP files instead:
    • 32-bit Dart Editor (latest)
    • 64-bit Dart Editor (latest)

  2. Unarchive the ZIP file by double-clicking it.

  3. Put the resulting directory somewhere convenient, such as under your Documents folder.

    We'll call this your Dart installation directory. This installation directory contains the DartEditor executable file and several subdirectories, including a samples directory.

    Important: Do not use the Dart installation directory for any other Eclipse-based software. If Dart Editor and Eclipse share the same workspace directory, they'll save conflicting settings, resulting in warnings and errors from both Dart Editor and Eclipse.

  4. If you don't already have a Java runtime, download and install it.

    • You can download the Java runtime environment (JRE) from Java SE Downloads.
    • To avoid having to modify the PATH environment variable, you can install the JRE under your Dart installation directory, in a subdirectory named jre.

Step 2: Launch the editor

Go to your Dart installation directory, and double-click DartEditor.

Step 3: Open a sample

The Dart Editor bundle comes with several samples. In this step, you'll open one and run it.

  1. Go to the File menu, and choose Open.... Or use the keyboard shortcut (Ctl+O or, on Mac, Option+O).

  2. Navigate to your Dart installation directory, then to samples, and then to sunflower. Double-click Sunflower.dart.

Your Dart Editor should now look something like this:

Dart Editor not only displays the contents of Sunflower.dart, but also updates the Libraries view to list important files for the sample. This list includes the resources that Sunflower.dart refers to, plus any HTML files in the Dart file's directory that refer to the app's code,

The Libraries view puts this list under a node called sunflower, corresponding to the following line of code in Sunflower.dart: 

 #library('sunflower');

The Libraries view displays entries for the Dart core and dom libraries. The core library is there because every Dart program uses it. The dom library is there because of this program's reference to it in the following statement: 

#import('dart:dom');

For more information about the Sunflower example, read the Sunflower Code Walkthrough.

Step 4: Run the sample

To run any program in Dart Editor, just click the Run button while any item in that program's library is selected. Dart Editor compiles your Dart program to JavaScript. It then brings up a browser window that displays the program's HTML page, with the JavaScript version of the program running inside it.

Tip: Sometimes the automatic compilation doesn't work. If you get a message that the JavaScript output was not generated, try using Tools > Force Recompile.

For example, to run Sunflower, click Sunflower.dart and then the Run button.

Step 5: Try some Dart Editor features

This section introduces you to some of Dart Editor's features, such as finding where APIs are declared, outlining the contents of a Dart file, and auto-completing references to APIs. We expect to add many more features over time.

  1. In the Editor view of Sunflower.dart, select anywhere within the string Math (at the top of the Sunflower() constructor). Then either press F3 or right-click and choose Open Declaration.

    The editor opens a new tab that displays the file that declares the Math type.

    Tip: You can also use this trick to find declarations within the same file.

  2. Click the left arrow button (to the left of the Run button) to go back to Sunflower.dart.

  3. Either press Alt+O (Option+O on Mac) or right-click and choose Quick Outline.

    A panel comes up displaying the classes, methods, and fields declared in Sunflower.dart. It looks something like this:


    You can reduce the size of the list by typing one or more characters. For example, if you type x, only the xc variable appears.

    Choose an item from the list—for example, xc. You'll be taken to its declaration in Sunflower.dart.

  4. Now it's time to use Dart Editor's auto-complete feature.

    1. Find the declaration of TAU, near the end of Sunflower.dart: 

      static final TAU = Math.PI * 2;

    2. Delete everything to the right of Math, and then type a period (.). After a short pause, a list of autocomplete suggestions comes up, looking something like this:

    3. Type or use the arrow keys to select PI, and then press Enter or Return to dismiss the panel.

    4. Complete your changes—for example, if you think sunflower seeds should be semi-circular, use Math.PI instead of Math.PI*2—and then save the file.

      If Dart Editor displays an error after you save, you've made a syntax error. Fix it or undo your typing, and then save again.

    5. Click the Run button to run your changed version.

    6. Back in Dart Editor, edit the Dart code again. This time use Ctl+Space to bring up the auto-complete suggestions.

      For example, type Mat, then Ctl+Space to see a list of classes and interfaces that start with "Mat".

Step 6: Create a new web app

Now that you're familiar with Dart Editor, you can create a web app from scratch.

  1. Click the New Application button (at the upper left of Dart Editor) to create a new app. Alternatively, choose File > New Application... from the Dart Editor menu.

    A dialog comes up, looking something like this:

  2. Type in a name for your application—for example, HelloWorld. If you don't like the default directory, type in a new location or browse to choose the location. Click Finish to create the initial files for the app.

    A default Dart file appears in the Edit view, and a few additional items appear in the Libraries view. Your Dart Editor window should look something like this:

  3. Click the button to run your new app.

  4. View the HTML file in Dart Editor by double-clicking it in the Libraries view.

    Note that the HTML file's <script> tag refers to a file that ends with .dart.app.js. You can't see this file in Dart Editor, but this JavaScript file is automatically created from your Dart code. Compiling to JavaScript allows browsers that know nothing about Dart to execute your app.

    Note: The generated JavaScript is rather huge right now. Don't worry, we'll fix that.

  5. Modify the default Dart and HTML files as you like, using auto-completion, samples, and API reference documentation to figure out which APIs you can use.
