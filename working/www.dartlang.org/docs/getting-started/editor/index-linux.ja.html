チュートリアル: Dart Editor (Linux)

このチュートリアルでは Linux で Dart Editor をインストールする方法を示します。

Dart Editor は Eclipse コンポーネント群を基にした軽量でオープンソースなエディターです。 Dart Editor で Dart プログラムの編集作業をすると、 Dart-to-JavaScript コンパイラーを実行したり、 Dart ベースのウェブアプリケーション群を起動したりもできます。

ノート: 以下の環境向けのイントールガイドもあります:
  • Windows
  • Mac

ステップ 1: ソフトウェアのダウンロードとインストール

このステップで Dart Editor と、必要に応じて Java の実行時環境もインストールします。

  1. システムアーキテクチャーに合った Dart Editor の ZIP ファイルをダウンロードします:
    • 32-bit Dart Editor
    • 64-bit Dart Editor

    使用している環境が 32-bit か 64-bit か判らない場合は uname -a コマンドを使います。 もしも表示に 64 が含まれていたなら、 それは恐らく 64-bit の Linux でしょうし、 そうでなければ 32-bit Linux でしょう。

    改版についてのノート: この Dart Editor のバージョンに関する情報は、 改版履歴 を見て下さい。 もし最新の未テストの Dart Editor を試してみたい場合には、 こちらの ZIP を使います:
    • 32-bit Dart Editor (最新版)
    • 64-bit Dart Editor (最新版)
  2. ZIP ファイルを展開します。例: 
     $ unzip DartBuild-linux.gtk.x86_64.zip

  3. 展開されたファイル郡をどこか適当で便利な場所に配置しましょう。例えば、 /home/me/dart. などに。

    この場所を Dart インストールディレクトリと呼ぶ事にしましょう。 このディレクトリには DartEditor の実行ファイルと幾つかのサブディレクトリ、またサンプル群などが含まれています。

    重要: Dart インストールディレクトリを他の Eclipse ベースのソフトウェアの為には 使用しないで下さい。 もし、 Dart Editor と Eclipse で 同じ workspace ディレクトリーを共有してしまうと、 それらの設定が衝突してしまい Dart Editor と Eclipse の両方で エラーや警告が報告される事でしょう。

  4. もし、 Java 実行時環境をまだ準備していない場合は、ダウンロードとインストールを済ませましょう。 JDK 7 は Java SE 開発キット 7 ダウンロード から入手できます。

ステップ 3: サンプルを開

Dart Editor には幾つかのサンプルが含まれています。このステップではそのうちの1つを開き、実行してみましょう。

  1. File メニューから Open... を選びます。 或いは、キーボードショートカットの (Ctl+O 、または Mac なら Option+O)を使っても良いでしょう。

  2. Dart インストールディレクトリへと辿り、 samples の、 sunflower へ。 そして、Sunflower.dartをダブルクリックします。

Dart Editor は次の様に表示されます:

Dart Editor は Sunflower.dart の内容のほか、サンプルでインポートしているファイルの一覧についての Libraries ビュー も更新します。このリストには Sunflower.dart の参照の他、 Dart ファイルのディレクトリにあるアプリケーションから参照される HTML ファイル群も含まれます。

Libraries ビュー には sunflower というノードが、 Sunflower.dartの次の行に対応して表示されます: 

 #library('sunflower');

Libraries ビューには Dart ライブラリの core と dom も表示されてます。 core ライブラリはあらゆる Dart プログラムで使用されます。 dom ライブラリはプログラムの次の命令文で参照されています: 

#import('dart:dom');

Sunflower についてのより詳細な情報については、 Sunflower Code Walkthroughを見て下さい。

ステップ 4: サンプルを動かす

Dart Editor からプログラムを起動したい場合は、 プログラムのライブラリを選択して、 Run ボタン をクリックします。 Dart Editor が Dart プログラムを JavaScript へコンパイルします。 その後、ブラウザーのウィンドウが起動し、 プログラムの HTML ページが表示され、 JavaScript版となったプログラムが動作します。

ヒント: 時折、自動コンパイルが動作しない事もあるかもしれません。 JavaScript の出力が生成されない旨のメッセージが示された際には、 Tools > Force Recompile を試して見て下さい。

Sunflower に於いて、 Sunflower.dart 、 Run ボタンをクリックした例を示します。

ステップ 5: Dart Editor の機能を幾つか試してみる

この節では Dart Editor の機能のうち、 API 宣言の検索、 Dart ファイルのアウトライン、 API 参照の自動補完について紹介します。より多くの機能の搭載については暫くお待ち下さい。

  1. Sunflower.dart の Editor ビューで、 何処か適当な所にある Math を選択して下さい (Sunflower() コンストラクターの最上部とか)。 それから F3 または右クリックしてOpen Declarationを選択します。

    エディターには新しいタブが開かれ、 Math 型の宣言が表示されます。

    ヒント: この方法を使って同じファイル内の宣言を見付け出す事もできます。

  2. （Run ボタンの左側にある）左矢印ボタン をクリックすると、 Sunflower.dart へ戻れます。

  3. 次に Alt+O (Mac では Option+O) または、 右クリックして Quick Outline を選びます。

    Sunflower.dart に措ける クラス群、メソッド群、フィールド宣言 についてのパネルが表示されます。 次の様に表示されるでしょう:


    文字を入力する事でリストからの絞り込みもできます。 例えば、 x と打ち込むと、 xc という変数だけが表示されるでしょう。

    リストから何か、例えば、 xc を選択して見ましょう。 Sunflower.dart に於けるその宣言が示されます。

  4. 次は、 Dart Editor の自動補完機能を見ましょう。

    1. Sunflower.dart の終端近くにある, TAU の宣言を見付けましょう: 

      static final TAU = Math.PI * 2;

    2. Math の右側を削除して、 ピリオドを打ち込みましょう (.)。 ちょっと待たされるかもしれませんが、 次に示す様な 自動補完リストによるお伺いが出現します:

    3. 打ち込むか、或いは矢印キーで PI を選択し、 Enter または Return キーを押すとパネルが閉じられます。

    4. 変更を終え &mdash 例えば、 Math.PI の換わりに Math.PI*2にして 向日葵の種を半円状にしてみようかな、とか — たならば、 ファイルを保存しましょう。

      保存後に Dart Editor がエラーを表示した場合は、 きっと文法エラーが含まれます。 修正するか、入力を元に戻して もう一度保存して見ましょう。

    5. Click the Run button to run your changed version.

    6. さて、 Dart Editor に戻り、 再び Dart コードを編集しましょう。 今度は Ctl+Space を使い、 自動補完によるお伺いを表示してみましょう。

      例えば、 Mat と打ち込んだ状態で Ctl+Space すると、 "Mat" で始まるインターフェースやクラスのリストが示されます。

ステップ 6: ウェブアプリケーションの新規作成

Dart Editor を使ってスクラッチからウェブアプリケーションを作成してみましょう。

  1. (Dart Editor の左上の方の) New Application ボタン をクリックし、 新しいアプリケーションを作成しましょう。 Dart Editor のメニューから File > New Application... と辿っても同じ事です。

    次の様なダイアログが表示されます:

  2. アプリケーションに適当な名前を付けてあげます — 例えば、 HelloWorld とか。 もしデフォルトのディレクトリが気に入らなければ、 配置先を適宜に指示すると良いでしょう。 Finish をクリックするとアプリケーションのための初期ファイル群が生成されます。

    デフォルトの Dart ファイルが Edit ビューに表示され、 Libraries ビューにも何やら幾つかの項目が表示されます。 Dart Editor ウィンドウは次の様になっている事でしょう:

  3. Run ボタン をクリックすると新しいアプリケーションが起動します。

  4. Libraries ビューでダブルクリックして、 Dart Editor に HTML ファイルを表示できます。

    この HTML ファイルの末尾の <script> タグは .dart.app.js を参照します（訳註: Dart Editor のバージョンによっては MIMEが application/dart のスクリプトが埋め込まれている可能性もある。）。 しかし、 Dart Editor でこのファイルを見ることはできません。 この JavaScript ファイルは Dart コードから自動的に生成されるものだからです。 これは Dart をブラウザーが実行できない場合に JavaScript へコンパイルする仕掛けです。

    ノート: 現時点では生成される JavaScript は非常に大きい有り様ですが、 これについては修正を予定しているのであまり心配しないで下さい。

  5. お好みで Dart や HTML ファイル群を 自動補完や、 サンプル群、 API リファレンス文書 などを使いながら編集し、 何ができるのか確かめてみて下さい。
