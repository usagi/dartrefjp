SDK

このページではどの様にして Dart SDK のプレリリース版を見付け、そして使えるのか教えます。SDKには以下のツール群が含まれます:

  • Dart-to-JavaScript コンパイラー (frog)
  • Dart 仮想マシン
  • Dart ライブラリー群

SDK についてのより詳細な情報はブログへの投稿 Dart SDK Quick Tourをご覧ください。

ノート: SDKはまだ若く、もしかしたら上手く動かないかもしれません。もし Dart をちょっと使ってみたいだけであれば、 Dartboard をお試し下さい。また、実際の開発環境としては Dart Editor をお使い下さい。

SDK の入手

SDKをまとめたリリースをダウンロード出来ます。これらは凡そ動作テストされています。

  • Windows SDK
  • Mac SDK
  • Linux SDK

或いは、危険は伴いますが、最新の生の未テストの SDK をダウンロードする事もできます。こちらは日毎、或いはもっと頻繁に、継続的にビルドされています。

  • Windows SDK (最新)
  • Mac SDK (最新)
  • Linux SDK (最新)

SDK のダウンロードと展開を終えたなら、 dart-sdk という名前のディレクトリがある事でしょう。この中には bin ディレクトリがあり、 frogc (Dart-to-JavaScript コンパイラー) と dart (Dart 仮想マシン)が含まれています。

frog を使う

bin/frogc を用いて Dart ファイルを JavaScript にコンパイルできます。 例: 

 $DART_SDK/bin/frogc
test.dart

このコマンドはあなたの Dart コードと等価な .js ファイルを生成します。

仮想マシンを使う

bin/dart を用いて Dart ファイルを Dart 仮想マシンで実行できます。 例: 

 $DART_SDK/bin/dart test.dart

型チェックの有効化

Dart プログラムは2つのモードの何れか1つで動作します: checked または production checked モードでは、動的チェックが割り当てられ、 実行時に一定の型システム違反について例外が通知がされます。 production モードでは、静的な型付けは無意味となります。

開発段階に措いては checked モードを使うことを推奨します。checked モードで実行する場合は次の様にコマンドラインで指示します: 

  $DART_SDK/bin/dart --enable_type_checks test.dart

アサーションの有効化

アサーションのチェックは真偽値で行い、 false であれば例外を通知します。 assert() 文は production モードでは動作しません。 開発中はアサーションを有効化する事を推奨します。

コマンドラインに次の様に指示する事でアサーションを有効化できます: 

  $DART_SDK/bin/dart --enable_asserts test.dart
